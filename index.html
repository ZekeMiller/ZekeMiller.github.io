Check

Check is my current primary passion project. It arose as a result of my desire to track as much of my own life as possible, my inability to find an app or service that did exactly what I wanted, and my absurd level of stubbornness. I decided that if I couldn't find one app I really liked, I should take all the aspects of all the things I've used, along with a lot of my own ideas, and combine them into one app. I figured it would be a good learning experience at the same time. My goal is to be able to track:

  • Habits, which are things you can do at any point, as many times a day as you want, such as "making a healthy diet decision". The purpose of a Habit is that you can do it as often as you want. Habits will have buttons for completion and failure. You can build a streak (similar to Snapchat) of days in which you do a Habit at least once, as well as days that you fail. The goal of course being to build up the success streak as long as possible and to not let a failure streak begin.
  • Dailies (which I need a new name for (possibly Repeats)), which are tasks that you want to complete on a regular schedule. This could be daily, weekly, monthly, on mondays, every other day, at least 3 times within a week, etc. I have a lot of plans for extremely flexible scheduling, since not all tasks are created equal (speaking of which, Tasks could be a good name for them). The purpose of whatever I end up calling these things is that you build a routine and implement the desired behavior in your normal schedule. I plan on also adding streaks to these. A perfect sample would be daily meditation.
  • Datas (I know, the names get worse as I go. I'm thinking of calling these Trackables), which are just things to put data into and graph easily. I'm the least certain on my goals for these, I just know I want some way of visualizing data (such as time spent sleeping).

I've implemented Habits, and both Dailies and Datas should go much more quickly now. Unfortunately I am extremely busy this semester, so my rapid summer development pace has slowed dramatically.

Rubik's Cube Solver Project

For several years now, 'cubing' has been a favorite hobby of mine. I own several rubik's cubes, as well as quite a few similar puzzles, such as a 2x2x2, a 4x4x4, a 5x5x5, a pyraminx, and a megaminx. In fact, I've started a rubik's cube related passion project. It started as me trying to figure out how to store a rubik's cube state, and how to modify it in predictable ways (individual turns). I then made a way for the program to recognize turns in Standard Notation as commands, perform them, and print the new cube position.

From there, I figured out how to map algorithm names to a string version of the list of names, so that the user can call an algorithm instead of individual moves. The biggest hurdle so far has been adding the functionality for the program to solve the cube, and tell the user exactly what moves to do to solve it as well. In order to compliment this, I added functionality for the user to input a scrambled cube, face by face, and on the condition that it is legally solvable, the program will solve it. One of the toughest parts of letting a user submit data on their own was to map the data to the structure I used to store positions and colors. I have plans to add some verification to check whether or not the cube is solvable, and tell the user if they have inputted an illegal position. The project is hosted at this GitHub page.

The project is somewhat abandoned, as I don't think any worthwhile changes are available. I wrote this project before I learned much of what I now know, including a lot of style decisions, as well as how to code in a language besides python. Besides, I have more than enough other interests to continue making other projects, and leave this one how it is.
