Rubik's Cube Solver Project

For several years now, 'cubing' has been a favorite hobby of mine. I own several rubik's cubes, as well as quite a few similar puzzles, such as a 2x2x2, a 4x4x4, a 5x5x5, a pyraminx, and a megaminx. In fact, I've started a rubik's cube related passion project. It started as me trying to figure out how to store a rubik's cube state, and how to modify it in predictable ways (individual turns). I then made a way for the program to recognize turns in Standard Notation as commands, perform them, and print the new cube position.

From there, I figured out how to map algorithm names to a string version of the list of names, so that the user can call an algorithm instead of individual moves. The biggest hurdle so far has been adding the functionality for the program to solve the cube, and tell the user exactly what moves to do to solve it as well. In order to compliment this, I added functionality for the user to input a scrambled cube, face by face, and on the condition that it is legally solvable, the program will solve it. One of the toughest parts of letting a user submit data on their own was to map the data to the structure I used to store positions and colors. I have plans to add some verification to check whether or not the cube is solvable, and tell the user if they have inputted an illegal position. All of is be hosted at this GitHub page.
